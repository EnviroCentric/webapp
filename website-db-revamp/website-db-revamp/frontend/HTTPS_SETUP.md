# HTTPS Setup for React Development

This project is now configured to use mkcert for local HTTPS development.

## What's been set up:

1. **mkcert installed**: A local certificate authority tool
2. **Local CA installed**: Your system now trusts certificates generated by mkcert
3. **SSL certificates generated**: Located in `frontend/certs/`
   - `localhost+3.pem` - Certificate file
   - `localhost+3-key.pem` - Private key file
4. **Vite configured**: Updated to use the mkcert certificates

## How to use:

### Option 1: Docker with HTTPS (Recommended)
```bash
docker-compose up --build frontend
```
Your app will run at: **https://localhost:5173**

### Option 2: Local development
```bash
cd frontend
npm run dev
```

### Option 3: Run with explicit HTTPS flag
```bash
cd frontend
npm run dev:https
```

## Benefits:

✅ **Trusted certificates**: No browser warnings  
✅ **Service Workers**: Can test PWA features  
✅ **Secure contexts**: APIs requiring HTTPS will work  
✅ **Real HTTPS testing**: Closer to production environment  

## Accessing your app:

- **https://localhost:5173** - Main access point
- **https://127.0.0.1:5173** - Alternative access  
- **https://192.168.1.237:5173** - Mobile access (your local network IP)
- The app is accessible from mobile devices on your local network

## Notes:

- Certificates are valid for 3 years (expires January 7, 2028)
- The certificates work for `localhost`, `127.0.0.1`, `::1`, and `192.168.1.237`
- Docker automatically mounts the certificates into the container
- Mobile camera access requires HTTPS - use https://192.168.1.237:5173
- Your backend also uses HTTPS at https://192.168.1.237:8000

## Troubleshooting:

If you get certificate errors:
1. Make sure mkcert CA is installed: `mkcert -install`
2. Regenerate certificates if needed: `mkcert localhost 127.0.0.1 ::1`
3. Restart your browser after installing the CA

## Switching back to HTTP:

If you need to switch back to HTTP for any reason, you can:
1. Comment out the `https` section in `vite.config.js`
2. Use `npm run dev` as before (it will default to HTTP)
